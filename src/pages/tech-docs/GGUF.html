GGUF：下一代 llama.cpp 量化模型格式

将模型权重、词表、超参数、量化元数据打包到统一文件中，支持 Q4/Q5/Q6 多种方案，方便桌面端与移动端推理。

特性

可扩展元数据、跨平台、支持所有 llama.cpp backend、增量下载。

量化类型

Q2_K、Q3_K、Q4_0、Q5_1、Q8_0、IQ3/4 等，平衡精度与体积。

部署

llama.cpp、llama.cpp server、KoboldCpp、LM Studio、Ollama。

文件结构

Header + Tensor Blocks + Metadata，支持分块校验。

📦 工作流

  • 使用 `convert.py` 将 HF 权重转为 GGUF，并选择量化方案。
  • 可指定 `--metal`/`--cuda` backend 以生成优化过的张量。
  • 推理端通过 mmap 方式加载 GGUF，避免全部解压到内存。
  • 可使用工具 `llama.cpp --dump-meta` 检查元数据。

📐 数学与编码

块量化

GGUF 使用固定大小 block，如 32/64 元素：

$$w_{block} = s \cdot q + m$$

其中 $s$ 为缩放，$q$ 为量化整数，$m$ 可选偏置。

内存映射

推理时直接 mmap：

$$\text{ptr} = \text{mmap}(\text{GGUF}, \text{PROT\_READ})$$

避免拷贝，降低启动时间。

💻 操作示例

转换 & 推理

# 1. 将 HF 权重转为 GGUF 并量化
python3 convert.py \
  --model llama-2-13b \
  --output llama-2-13b.q4_k.gguf \
  --quant q4_k

# 2. 使用 llama.cpp 运行
./main -m llama-2-13b.q4_k.gguf -p "你好, 请介绍量化"
