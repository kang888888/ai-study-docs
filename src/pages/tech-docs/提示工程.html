提示工程（Prompt Engineering）

设计和优化提示以获得更好生成结果的技术，是文本生成的核心技术之一。

核心特点

提示设计、上下文管理、示例选择、迭代优化。

提示类型

零样本、少样本、思维链、角色提示。

设计原则

清晰性、一致性、完整性。

应用场景

文本分类、生成、问答、代码生成等。

⚙️ 提示类型

  • 零样本提示：不提供示例，直接描述任务
  • 少样本提示：提供少量示例指导模型
  • 思维链提示：引导模型进行逐步推理
  • 角色提示：让模型扮演特定角色

🔧 主要技术

1. 零样本提示（Zero-shot Prompting）

  • 特点：直接描述任务，不提供示例
  • 优势：简单快速，适用于常见任务
  • 劣势：对复杂任务效果有限

2. 少样本提示（Few-shot Prompting）

  • 特点：提供少量示例展示期望行为
  • 优势：效果好，适应性强
  • 劣势：需要精心选择示例

3. 思维链提示（Chain-of-Thought）

  • 特点：引导模型展示推理过程
  • 优势：提升复杂问题解决能力
  • 劣势：增加token消耗

情感：

英文：Thank you

中文：再见
英文：

1. 开始有10个苹果
2. 吃了3个，剩余：10 - 3 = 7个
3. 买了5个，现在有：7 + 5 = 12个

答案：12个

📊 设计原则

  • 清晰性：使用清晰、具体的指令，避免歧义
  • 一致性：保持格式、术语、风格的一致性
  • 完整性：提供完整信息、必要约束和预期结果
